## 2. 结构范式

### 2.1 Frontmatter-Body 范式

- **描述**：YAML 头 + Markdown 体的标准结构
- **示例**：`self-improving-agent.md`、`agent-browser.md`
- **优势**：元数据丰富、易于解析、工具友好
- **劣势**：需要 YAML 解析、学习成本稍高
- **质量评估**：可维护性 ⭐⭐⭐⭐ | 可演进性 ⭐⭐⭐⭐ | 可复用性 ⭐⭐⭐⭐⭐

### 2.2 标准章节划分

- **描述**：目标 → 输入 → 执行步骤 → 输出 → 注意事项
- **示例**：多数 Skill 遵循此模式
- **优势**：逻辑清晰、易于理解
- **劣势**：灵活性不足
- **质量评估**：可维护性 ⭐⭐⭐⭐⭐ | 可演进性 ⭐⭐⭐ | 可复用性 ⭐⭐⭐⭐

## 3. 内容范式

### 3.1 Goal-Steps-Output 范式

- **描述**：目标 → 步骤 → 输出的线性流程
- **示例**：`collect-skills.md`
- **优势**：逻辑清晰、符合思维习惯
- **劣势**：难以表达复杂逻辑
- **改进建议**：增加决策分支和异常处理章节

### 3.2 Quick Reference 范式

- **描述**：快速参考表 + 详细说明
- **示例**：`self-improving-agent.md`
- **优势**：查找方便、易于使用
- **劣势**：占用空间大
- **适用场景**：工具类、参考类 Skill

### 3.3 命令列表范式

- **描述**：命令 + 示例 + 说明
- **示例**：`agent-browser.md`
- **优势**：操作性强、易于复制
- **劣势**：缺少上下文
- **适用场景**：CLI 工具类 Skill

## 4. 交互范式

### 4.1 命令触发范式

- **描述**：通过 `/xxx` 命令显式触发
- **示例**：所有收集的 Skill
- **优势**：意图明确、易于记忆
- **劣势**：需用户主动触发
- **适用场景**：工具类、工作流类 Skill

### 4.2 条件触发范式

- **描述**：满足条件自动执行
- **示例**：`self-improving-agent.md`（失败时自动触发）
- **优势**：智能便捷
- **劣势**：可能误触发
- **适用场景**：监控类、自动化类 Skill

## 6. 最佳实践总结

1. **结构标准**：使用 YAML frontmatter + 标准章节
2. **目标明确**：定义清晰的使用时机和目标
3. **步骤适中**：粗粒度概述 + 关键细节展开
4. **示例丰富**：提供可直接复制使用的示例
5. **错误处理**：说明常见错误和处理方式
6. **快速参考**：复杂 Skill 提供速查表
7. **版本管理**：长期维护的 Skill 记录变更
